# Пояснительная записка к учебной практике
## Информационная система управления конвейерным производством

**Стек:** Avalonia UI, C#, Entity Framework Core, PostgreSQL  
**База данных:** `BaseForYchebka`

---

## 1. Планировка цехов (задание 2.1)

### 1.1. Задача по методичке

Директор должен иметь возможность открыть чертёж (план) цеха и расставить на нём значки: оборудование, выход, огнетушитель, аптечку. Расположение значков сохраняется в базе данных.

На занятиях мы не проходили интерактивные схемы «перетащи значок на план», поэтому реализация сделана на основе стандартных возможностей Avalonia: **Canvas**, **Image** и обработчиков мыши.

### 1.2. Как устроено хранение данных

| Таблица | Назначение |
|---------|------------|
| `Workshop` | Список цехов. Поле `PlanImage` (bytea) — PNG/JPG самого чертежа цеха |
| `WorkshopPlanItem` | Значки на плане: тип (`IconType`), координаты `PosX`, `PosY`, связь с цехом |

Чертежи цехов из архива методички загружаются в БД **один раз** скриптом `sql-export/import_workshops.py`:
- из zip-архива извлекаются PNG-файлы планов;
- каждый файл записывается в таблицу `Workshop`.

Значки (иконки оборудования, выхода и т.д.) лежат в папке проекта `WorkshopAssets/icons/` и копируются в выходную папку при сборке (настройка в `.csproj`).

### 1.3. Как работает экран «Планировка цехов»

**Файлы:** `Pages/WorkshopPlanningControl.axaml` + `.axaml.cs`  
**Доступ:** роль **Директор** → кнопка «Планировка цехов».

**Алгоритм работы формы:**

1. При открытии из БД загружается список цехов (`Workshop`).
2. Пользователь выбирает цех в `ComboBox`.
3. Из поля `PlanImage` создаётся картинка (`Bitmap`) — это **фон** на `Canvas`.
4. Из таблицы `WorkshopPlanItem` читаются сохранённые значки и рисуются поверх фона в координатах `(PosX, PosY)`.
5. Слева — панель инструментов с 4 типами значков.
6. **Добавление значка:** нажать на значок слева → отпустить кнопку мыши на плане → в этой точке появляется иконка.
7. **Перемещение:** нажать на значок на плане и перетащить мышью.
8. **Удаление:** выбрать значок на плане → «Удалить значок».
9. **Сохранить:** старые записи `WorkshopPlanItem` для цеха удаляются, текущие координаты всех значков записываются заново.
10. **Отменить:** восстанавливается состояние на момент последней загрузки цеха (резервная копия в памяти).
11. **Увеличить / Уменьшить:** масштабирование через `ScaleTransform` (удобно на больших чертежах).

**Почему Canvas, а не PictureBox:** в Avalonia для произвольного размещения элементов на изображении используется `Canvas` с методами `Canvas.SetLeft` / `Canvas.SetTop`. Это аналог «наложения объектов на картинку» в WinForms, но в стиле Avalonia.

### 1.4. Что показать преподавателю на защите (планировка)

1. Войти как **Директор**.
2. «Планировка цехов» → выбрать цех (например, «Сборочный»).
3. Перетащить на план значок «Оборудование» и «Огнетушитель».
4. Нажать **Сохранить**.
5. Выйти из формы и зайти снова — значки остались на местах (данные в PostgreSQL).
6. Нажать **Отменить** после случайного сдвига — план вернётся к сохранённому виду.

## 2. Загрузка файлов в базу данных

В приложении файлы не хранятся на диске пользователя — они сохраняются в PostgreSQL в полях типа **bytea** (массив байтов).

### 2.1. Где используется загрузка

| Место | Таблица | Поля | Кто загружает |
|-------|---------|------|---------------|
| Оформление заказа | `OrderDrawing` | `FileName`, `FileData` | Заказчик, Менеджер |
| Спецификация изделия (раздел «Чертежи») | `ProductDrawing` | `FileName`, `FileData` | Мастер |

### 2.2. Как это реализовано в коде (общий принцип)

Используется стандартный диалог Avalonia — `StorageProvider.OpenFilePickerAsync`:

```csharp
var files = await topLevel.StorageProvider.OpenFilePickerAsync(...);
await using var stream = await file.OpenReadAsync();
using var memory = new MemoryStream();
await stream.CopyToAsync(memory);
// memory.ToArray() → byte[] → поле FileData в Entity Framework
```

**Шаги:**
1. Пользователь нажимает «Добавить файл».
2. Открывается системный диалог выбора файла.
3. Файл читается в память (`byte[]`).
4. При сохранении заказа/спецификации массив байтов записывается в PostgreSQL через EF Core.

**Пример для заказа** (`CreateOrderControl.axaml.cs`):
- чертежи сначала попадают во временный список `newDrawings`;
- при нажатии «Сохранить» создаётся запись `OrderDrawing` с именем файла и содержимым.

**Пример для спецификации** (`SpecificationControl.axaml.cs`, раздел «Чертежи»):
- при «Добавить» сразу создаётся запись `ProductDrawing` в БД.

Удаление чертежа заказа при редактировании — запись удаляется из `OrderDrawing`. Удаление из спецификации — аналогично из `ProductDrawing`.

### 2.3. Что показать преподавателю (файлы)

1. **Заказчик** → «Заказы» → «Добавить заказ» → «Добавить файл» → выбрать любой PDF или PNG → «Сохранить».
2. Объяснить: файл в БД, не в папке проекта (можно показать таблицу `OrderDrawing` в pgAdmin, если спросят).
3. **Мастер** → «Изделия» → спецификация → раздел «Чертежи» → добавить файл.

## 3. Печать отчётов

### 3.1. Подход

В учебном проекте «печать» реализована как **формирование текстового отчёта** и **сохранение в файл `.txt`** через системный диалог «Сохранить как». Это проще системного принтера, но полностью закрывает требование «вывод на печать» — отчёт можно открыть и распечатать из блокнота или Word.

### 3.2. Общий компонент

**Файлы:** `Pages/PrintReportControl.axaml` + `.axaml.cs`

Любая форма собирает текст отчёта в `StringBuilder`, затем открывает:

```csharp
new PrintReportControl("Заголовок", reportText, () => new ...Control());
```

На экране — заголовок и многострочное поле с текстом. Кнопка **«Сохранить в файл»** вызывает `SaveFilePickerAsync` и записывает текст в выбранный `.txt`.

### 3.3. Где доступна печать

| Экран | Кнопка | Содержимое отчёта |
|-------|--------|-------------------|
| Спецификация изделия | «Печать» | размеры, материалы, комплектующие, сборочные единицы, операции |
| Расчёт заказа | «Печать» | потребности, себестоимость, сроки, список операций Гantt |
| Отчёт по складу | «Печать» | остатки по складам (поставщикам), итоги |

### 3.4. Что показать преподавателю (печать)

1. Менеджер → заказ → «Расчёт заказа» → «Печать» → «Сохранить в файл» → открыть сохранённый `report.txt`.
2. Директор → «Отчёт по складу» → «Печать» → сохранить.

### Шаг 1. Заказчик создаёт заказ (сессия 2.3)

1. Войти как **Заказчик**.
2. «Заказы» → **«Добавить заказ»**.
3. Заполнить:
   - Наименование: `Демо-заказ конвейера`
   - Изделие: выбрать из списка (например, готовый продукт из БД)
   - Количество: `1`
   - Описание: `Тестовый заказ для защиты`
4. **Размеры:** добавить, например: `Длина` / `12` / `м`
5. **Чертежи:** «Добавить файл» → выбрать любой файл.
6. **Сохранить** — появится автоматический номер заказа (12 символов: ФИ + дата + порядковый номер).
7. В списке заказов выбрать заказ → **«Редактировать»** (доступно только для статуса «Новый») → изменить описание → сохранить.

**Что сказать:** «Заказ в статусе „Новый“, его можно редактировать и удалять. Чертежи и размеры хранятся в БД.»

---

### Шаг 2. Менеджер принимает заказ (сессия 2.2)

1. Выход → войти как **Менеджер**.

4. Показать **историю статусов** справа в карточке заказа.

### Шаг 3. Конструктор указывает стоимость (сессия 2.2)

### Шаг 4. Мастер ведёт спецификацию (сессия 3.1)

2. «Изделия» → выбрать изделие из заказа → **«Спецификация»**.
3. Пройти разделы: Размеры, Материалы, Комплектующие, Операции (тип оборудования, время, описание).
4. Раздел **«Чертежи»** → добавить файл.
5. **«Печать»** → сохранить спецификацию в `.txt`.

### Шаг 5. Менеджер — расчёт и Гantt (сессия 3.2–3.4)

1. Войти как **Менеджер**.
2. «Заказы» → выбрать заказ → **«Расчёт заказа»**.
3. Показать:
   - список материалов/комплектующих, нехватку, себестоимость;
   - срок доставки;

   - итоговое время (доставка + производство).

## 5. Структура проекта (кратко для ответа на вопросы)

```
YchebnayaPractika/

  App.axaml.cs        — статический App.DbContext
```

## 6. Типичные вопросы преподавателя и короткие ответы

**В: Почему планировка цехов — не в Paint, а в программе?**  
О: По заданию 2.1 директор должен расставлять значки на плане и сохранять в БД. Реализовано на Canvas в Avalonia: фон из `Workshop.PlanImage`, значки — записи `WorkshopPlanItem` с координатами.

**В: Где хранятся чертежи заказа?**  
О: В таблице `OrderDrawing`, поле `FileData` (bytea). Имя файла — в `FileName`.

**В: Это настоящая печать?**  
О: Формируется текстовый отчёт и сохраняется в файл через диалог ОС. Его можно распечатать из любой программы. Для учебного проекта этого достаточно; системный PrintDialog можно добавить отдельно.

**В: Как считается номер заказа?**  
О: 12 символов: первая буква фамилии + первая буква имени заказчика + ГГГГММДД + порядковый номер (01–99, потом снова 01). Реализация в `CreateOrderControl.GenerateOrderNumber`.

**В: Как работает списание материалов?**  
О: В `OrdersControl` при смене статуса на «Производство» вызывается `WriteOffMaterials`: рекурсивный обход спецификации через `CollectNeeds` (материалы, комплектующие, дочерние изделия).
